## Codebase Patterns
- 后端使用 `uv` 管理依赖，开发命令 `cd backend && uv sync --extra dev`；启动 `uv run uvicorn app.main:app --reload --port 8000`
- 后端路径解析基于 `app/core/config.py` 中的 `PROJECT_ROOT`（`Path(__file__).resolve().parents[3]`），所有 workspace 路径常量集中在此
- workspace 目录在运行时通过 `app/core/workspace.py` 的 `ensure_workspace()` 自初始化（lifespan 事件触发），无需手工创建或提交 runtime 文件
- FastAPI 使用 application factory 模式 `create_app()`，便于 TestClient 测试
- CORS 白名单为 `http://localhost:5173`（前端 Vite 默认端口）
- 根 `.gitignore` 忽略整个 `workspace/`，workspace 内的 `.gitignore` 由 `ensure_workspace()` 在运行时生成
- 前端使用 `npm` 管理依赖，开发命令 `cd frontend && npm install && npm run dev`；构建 `npm run build`；类型检查 `npm run typecheck`；lint `npm run lint`
- 前端技术栈：React 18 + Vite 5 + TypeScript + Tailwind CSS 3 + react-router-dom v6 + lucide-react 图标
- 前端路径别名 `@/` 指向 `src/`，在 `vite.config.ts` 和 `tsconfig.app.json` 中同步配置
- Vite dev server 配置了 `/api`、`/docs`、`/openapi.json` proxy 代理到后端 `http://localhost:8000`，开发联调无需关心 CORS
- 暖色调主题定义在 `tailwind.config.js` 的 `colors.warm` 下（bg 白/menu 浅灰/orange 暖橙/amber 琥珀）
- 前端布局：`src/components/Layout.tsx` 为左侧菜单 + 右侧内容区骨架，路由配置在 `src/App.tsx`，占位页面统一用 `src/pages/PlaceholderPage.tsx`
- 新增后端 API 模块：在 `app/api/<feature>.py` 定义 `APIRouter(prefix="/<feature>")`，在 `app/api/router.py` 用 `api_router.include_router(...)` 挂载，自动获得 `/api/<feature>` 前缀
- 后端业务逻辑分三层：`app/schemas/`（Pydantic 校验）、`app/services/`（业务+持久化）、`app/api/`（路由编排）
- 文件持久化用 `filelock.FileLock` 包裹读写，锁文件放同目录 `<name>.json.lock`；workspace/ 被 gitignore
- keyring 密钥链可用性判断：`getattr(backend, "name", "")` 含 "fail"/"null" 即不可用，降级到 env 引用；测试用 `keyring.set_keyring(MemoryKeyring())` 注入内存 backend
- 内置工具集固定为 `read_file`/`write_file`/`execute_command` 三元组（不可增删），启用状态存 `workspace/config/tools.json`，缺失默认 enabled=True；安全拦截统一抛 `SecurityBlockedError`（reason 字段回传给 Agent）
- 路径安全统一用 `app/core/path_security.py` 的 `is_safe_workspace_path`（`..` 拒绝 + 符号链接逃逸检测 + workspace 前缀校验）；工具函数中相对路径需先锚定到 `WORKSPACE_DIR` 再校验
- `execute_command` 安全护栏：黑名单子串匹配(大小写不敏感)、`subprocess.run(timeout=60)` 自动 SIGKILL、working_dir 限定 workspace/、输出截断 100KB、`_strip_sensitive_env` 剥离 `OPENAI_API_KEY`/`MINI_WORKBUDDY_*`/`*_TOKEN` 等敏感环境变量
- 测试隔离陷阱：`workspace/config/tools.json`（工具启用态）**不在** `agents_store.reset_for_test()`/`conversations_store.reset_for_test()` 的清理范围内，且 workspace/ 被 gitignore 不会随 commit 重置。任何依赖工具启用态的测试（如 agent-loop 请求体断言）必须在 setup 里显式调 `tools_store.reset_for_test()`，否则上一次测试遗留的 disabled 标志会静默改变后续测试行为（非确定性失败）
- SSE 流式端点 pattern：`StreamingResponse(async_gen, media_type="text/event-stream")` + `Cache-Control: no-cache` + `X-Accel-Buffering: no`（禁 nginx 缓冲）；事件统一 `data: {json}\n\n` 单行格式（见 `app/services/sse_events.py`），心跳为 `: heartbeat\n\n` 注释行；断连检测用 `await request.is_disconnected()` 轮询 + 写已关闭连接会抛异常双重保险
- Agent Loop（`app/services/agent_loop.py`）自解析 OpenAI 流式 `data:` 行（不依赖 OpenAI SDK），工具调用 delta 按 `index` 累积合并；每轮 user/assistant/tool 事件即时 `conversations_store.append_event` 落 JSONL，断连也保留已写记录
