## Codebase Patterns
- Frontend currently uses Lovable TanStack Start UI with mock-store; real APIs live under backend FastAPI `/api`.
- Prefer `frontend/src/lib/api/*` mappers for snake_case ↔ camelCase; do not persist API keys in localStorage.
- Backend model config id is UUID `id`; supplier model name must be stored in field `model`. Agent Loop must call chat completions with `model`, not display `name`.
- Builtin tools are fixed: `read_file` / `write_file` / `execute_command`.
- Skills are directories under `workspace/config/skills/{id}/SKILL.md`; management API is still missing at start of this feature.
- Chat send is SSE via `POST /api/chat/send`; frontend must stream with fetch and map events to UI parts.
- Only edit `frontend/`, not `frontend-bak/`.
- Keep API payload conversion in a domain facade under `frontend/src/lib/api/`; pages should use mapped ViewModels and never retain server-backed configuration in a mock store.
- Radix `DialogContent` should include a concise `DialogDescription` to prevent accessibility warnings in the browser console.
- When browser-testing header actions, inspect and dismiss persistent Sonner notifications first: they can intercept pointer clicks even when the intended button is enabled.
- Rebuild chat UI state from persisted JSONL `role`/`type`/`data` records, and merge live SSE `tool_result` events by `tool_call_id`, so refresh and streaming show the same interaction history.
- Agent configuration spans CRUD metadata and the separate `agent-md` endpoint; persist both when editing so model/tool bindings and the system prompt remain consistent.
- For browser closed-loop checks, use a local OpenAI-compatible SSE mock and ask it to write under `conversations/{conversation_id}/outputs/`; this makes the persisted download visible after refresh.
- 在此容器中启动 `agent-browser` 需要 `--args "--no-sandbox"`；若 daemon 已在运行，先执行 `agent-browser close` 才能使该参数生效。
- Shared dev-server workspace state can be reset by test processes; browser verification should query `/api/models`, `/api/agents`, and `/api/tools` immediately before sending a message.
